05 · Where We Came From, Where We're Going

Live Sheet vs in-app books.

The clinic ran for three years on a 13-tab spreadsheet. What follows is the objective delta — what each surface gives you, side by side.

Live Sheet today Phase 12 in-app books
Income statement Complete · 2024–2026 Generated from GL · period-over-period
Balance sheet Empty template tab Live · ties to GL
Cash flow statement Empty template tab Direct method · live
HMO AR per provider Tracked but data quality issues (#REF!, blanks) Reconciled subledger + aging + stuck-claim detector
Unbilled HMO detection Manual / by memory Automated report (12.3)
Doctor compensation Per-day PF noted; rent payments inline pf / rent / none classification · rent billing tracker
Payroll Out of scope · external ZKTeco DTR · semi-monthly · payslip PDF
Period close Implicit · no lock Quarterly · DB-enforced lock · reopen with reason
Audit trail Sheet revision history only audit_log on every write · queryable
VAT capture Lumped in “Legal & Regulatory” Deferred to Future Phase 1 (pending classification)
Key takeaway
The income statement is the only complete artifact in the Sheet. Everything else is incomplete or aspirational. The in-app build inherits the P&L shape verbatim, then fills the gaps the Sheet never closed — starting with HMO AR.

08 · Risks & Dependencies

What could go sideways, and how we handle it.

None of these are critical right now. All have a mitigation plan and a named owner.

MCP doesn't honor BEGIN/ROLLBACK Live · mitigated

Supabase MCP execute_sql auto-commits statements, so the 12.1 smoke leaked 6 JE headers into production despite the BEGIN; … ROLLBACK; envelope. Cleaned up post-smoke.

Mitigation · for 12.2+, run smoke against local Docker Supabase, OR have the smoke explicitly clean up its own rows after each assertion.
No local Supabase available Accepted

Docker not installed on the dev machine; migrations apply directly to the production project. Additive-only changes are safe; the user already works this way for past phases.

Mitigation · accept current pattern; add a documented option to install Docker before high-risk migrations (e.g., destructive 12.A backfill commits).
Sheet schema drift across 3 generations 12.A / 12.B

The mastersheet has at least three column generations of the same transaction (older “HMO INVOICE/*” → mid “HMO/STATUS” → newer “VERITAS PAY/*”). The import parser must normalize all three.

Mitigation · staged import with dry-run gate; data-quality report surfaces drift before commit; manual reclassification UI from 12.7.
Historical data quality 12.A

#REF! errors in BASE PRICE on some 2023 rows, HMO=YES with STATUS=N/A, blank submission dates against “SUBMITTED” status. Every such row needs a human decision.

Mitigation · 12.A's import is staged; admin reviews and resolves each finding before the commit.
VAT classification deferral Tracked

Schema stays extensible. Output VAT Payable, Input VAT, etc. can be added later without migration pain. Risk: a long deferral lets the audit gap (BIR remittances mis-classified as expenses) compound.

Mitigation · accountant classifies revenue lines as part of any quarter close after 12.7 ships.
Payroll compliance teeth 12.6

Withholding tax on compensation is legally mandatory; SSS, PhilHealth, Pag-IBIG split + employer contributions must be captured per run. Filing surfaces deferred but capture cannot be.

Mitigation · 12.6 includes capture; FP2 adds the filing hand-off (2316, alphalists).

11 · Key Takeaways

Three things to keep in mind.

If you remember nothing else from this document, remember these.

01

The GL is the contract.

Every operational event from 12.2 onward posts exactly one JE. Debits = credits is enforced at the database, period locks are DB-enforced. The app can never disagree with itself again.

02

The AR answer arrives early.

12.1 + 12.2 + 12.3 + 12.A delivers the unbilled-claim audit and the reconciled per-provider AR balance by roughly week four. That's the single most valuable artifact of the project.

03

Reception's life doesn't change.

The whole project sits behind the same admin gate. The forms reception fills out today are unchanged. The GL posts behind them silently. You're rebuilding the books, not the clinic's daily workflow.

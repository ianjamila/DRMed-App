Phase 12.4 follow-up · UI polish proposal v2

DRMed — UI/UX swap preview

Five candidate swaps from hand-rolled Tailwind to shadcn/ui components. All "After" panels assume the token bridge (Option A) below is in place — so the brand carries through automatically.

⓪ Prerequisite — Token bridge (one edit, ~10 lines)

Today shadcn's tokens (--primary, --card, etc.) point at grayscale defaults. Your existing shadcn pieces (Dialog, MFA enroll Button, etc.) currently render off-brand. One globals.css edit fixes the existing mismatch AND makes every subsequent swap inherit the brand for free. 

/* globals.css — add to :root */
--primary:               var(--color-brand-navy);     /* #284570 */
--primary-foreground:    #ffffff;
--ring:                  var(--color-brand-cyan);     /* focus rings: brand cyan */
--accent:                var(--color-brand-bg);       /* hover surfaces */
--accent-foreground:     var(--color-brand-text);
--muted:                 var(--color-brand-bg);
--muted-foreground:      var(--color-brand-text-soft);
--card-foreground:       var(--color-brand-text);
--destructive:           #dc2626;                       /* red stays universal */

Side effect: existing shadcn components (MFA enroll Button, Dialog titles in the recurring + bill-detail modals) snap to brand. Looking at the current state, that's an improvement — they're grayscale today.

Used everywhere. 50+ call sites. Most pervasive change.

5. Form input → Input + Label

Brand inheritance: high on focus state (bridge sets --ring to brand-cyan, so focus rings glow cyan). Label style shifts from "tiny uppercase muted" to "sentence-case medium" — that's a deliberate shadcn pattern, more legible.

Used in: vendor-form, bill-form, payment-form, recurring-client, MFA enroll, etc. 30+ call sites.
